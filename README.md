Проект: Работа с базой данных SQLite
Описание проекта
Данный проект демонстрирует полный цикл работы с базой данных SQLite в Python, включая создание БД, выполнение CRUD операций, работу с файлами и сложные SQL-запросы.

Выполненные действия
1. Создание и настройка базы данных
Что сделано:

Создано подключение к базе данных SQLite (database.db)

Создана таблица users с полями:

id - первичный ключ с автоинкрементом

name - текстовое поле для имени пользователя

email - уникальное текстовое поле для email

Ключевые моменты:

Использован PRIMARY KEY AUTOINCREMENT для автоматической генерации ID

Поле email помечено как UNIQUE для обеспечения уникальности

Применен COMMIT для сохранения изменений в БД

2. Реализация операций CRUD
CRUD - базовые операции с данными:

CREATE (Создание):

Добавление новых пользователей в таблицу users

Использование параметризованных запросов для безопасности

READ (Чтение):

Получение всех записей из таблицы

Использование fetchall() для получения результатов

UPDATE (Обновление):

Изменение существующих записей

Обновление email пользователя по имени

DELETE (Удаление):

Удаление записей из таблицы

Удаление пользователя по имени

3. Работа с файлами CSV
Импорт данных:

Создан тестовый CSV файл data.csv с данными пользователей

Реализовано чтение CSV файла и импорт данных в БД

Использован модуль csv для работы с CSV форматом

Экспорт данных:

Экспорт всех данных из таблицы users в файл export.csv

Сохранение в формате, пригодном для дальнейшего использования

4. Работа с файлами JSON
Импорт данных:

Создан тестовый JSON файл data.json со структурой данных

Реализован парсинг JSON и импорт в базу данных

Преобразование JSON-объектов в записи БД

Экспорт данных:

Экспорт данных из БД в JSON формат

Автоматическое преобразование структуры таблицы в словари

Сохранение с красивым форматированием (indent=2)

5. Сложные SQL-запросы
Создание связанных таблиц:

Создана таблица posts для хранения постов пользователей

Установлена связь через внешний ключ user_id

Выполненные запросы:

JOIN запрос:

Объединение таблиц users и posts

Подсчет количества постов для каждого пользователя

Использование GROUP BY для агрегации данных

Поисковые запросы:

Поиск пользователей по шаблону имени

Использование LIKE с подстановочными знаками

Запросы с сортировкой:

Сортировка результатов по ID в обратном порядке

Ограничение вывода с помощью LIMIT

Технические особенности
Безопасность:

Использованы параметризованные запросы для предотвращения SQL-инъекций

Правильное экранирование специальных символов

Управление соединениями:

Корректное открытие и закрытие соединений с БД

Использование COMMIT для подтверждения транзакций

Обработка данных:

Преобразование данных между разными форматами (CSV, JSON, SQL)

Работа с различными типами данных Python и SQLite
